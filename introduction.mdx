## What is SFLUV?

SFLUV is a local currency platform that uses a wrapped HONEY token on the Berachain blockchain. It's designed to strengthen local economies by connecting merchants, community improvers, project proposers, voters, and credential issuers in a multi-role governance and commerce ecosystem.

Think of SFLUV as a complete community workflow platform where:
- **Merchants** accept the local currency for goods and services
- **Proposers** suggest community improvement projects
- **Improvers** complete projects and earn currency
- **Voters** approve or reject proposed workflows
- **Issuers** grant credentials and certifications
- **Supervisors** oversee workflow completion and quality

### Democratic Workflow Approval

All proposed community projects go through a democratic voting process:
- **50% quorum requirement** - Half of eligible voters must participate
- **24-hour countdown** - Once quorum is reached, voting continues for 24 hours
- **Early finalization** - If more than 50% of all voters agree before countdown ends, workflow is approved immediately
- **Budget protection** - Workflows are blocked if funding is insufficient

### Credential System

Build trust and expertise through credentials:
- Workflow roles can require specific credentials
- Issuers grant credentials to qualified improvers
- Two built-in credential types: `dpw_certified` and `sfluv_verifier`
- Admins can create custom credential types

### Recurring Workflows

Set up repeating community projects:
- **Daily, weekly, or monthly** recurrence options
- Automatic series management with `series_id`
- Sequential blocking - new instances wait until previous workflow is paid out
- Improvers can claim entire series for consistent assignments

### Sequential Workflow Steps

Complex projects are broken into manageable steps:
- Steps unlock sequentially as previous steps complete
- Each step can be assigned to different improver roles
- Detailed work items with photo requirements, written responses, and dropdown options
- Camera-only capture options for authenticity

## Platform Architecture

<Note>
SFLUV is built on modern web and blockchain technologies to ensure security, transparency, and scalability.
</Note>

The platform consists of:

- **Frontend** - Next.js 15 with React 19, built with Tailwind CSS and Radix UI components
- **Backend** - Go 1.24 API server with PostgreSQL databases and JWT authentication via Privy
- **Blockchain Indexer** - Ponder service that monitors Berachain ERC20 events in real-time
- **Three Databases** - Separate databases for application data, faucet/bot operations, and blockchain events

All authentication uses Privy wallet integration, ensuring secure access while maintaining user control over their blockchain identity.
